Simon Scrapes walks through a dozen one-line Claude Code settings and slash-command habits meant to reduce waiting, permissions friction, token waste, telemetry, context rot, and unwanted attribution. The most actionable advice is to turn on notifications, tune permissions with explicit allow and deny rules, use stronger models for planning and cheaper models for execution, compact before context quality degrades, and keep publishing authority in your hands.

What this video offers

Turn on local notifications so long-running Claude Code tasks do not sit unnoticed at a permission prompt.

Use /fewer permission prompt to generate a tailored allow list from your actual history instead of blindly approving everything.

Add deny rules for .env reads and git push so secrets and publishing remain protected even when other permissions are loosened.

Keep high-effort and expensive models for planning, review, and difficult reasoning, then use faster or cheaper models for implementation.

Set cleanupPeriodDays above the default 30 days if you want to preserve conversation history.

Lower auto-compaction to around 75 percent so compaction happens before context rot becomes severe.
