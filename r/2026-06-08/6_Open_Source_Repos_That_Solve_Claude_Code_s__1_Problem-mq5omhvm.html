TL;DR
Duncan Rogoff rounds up 6 free repos aimed at the biggest Claude Code pain point: token cost. The core theme is better visibility, tighter context, shorter outputs, and smarter retrieval so Claude reads less, wastes fewer tokens, and gets to the right code or notes faster.
How These Repos Cut Claude Cost
Creator's Key Callouts
  • Check `/usage` before changing plans so you know where spend is actually going.
  • CC usage is framed as the visibility tool: useful for per-session, per-agent, and per-model tracking, plus reporting and a status bar.
  • RTK is presented as the biggest direct token saver by compressing bash command output before Claude sees it.
  • Caveman Claude is a favorite pick because shorter, stripped-down replies can cut token use and speed things up without hurting accuracy.
  • Andrej Karpathy's skills repo is recommended less for direct compression and more for preventing waste from bad assumptions, overbuilding, and unnecessary file edits.
  • Graphify is pitched as especially valuable for larger codebases because it helps Claude retrieve only the relevant parts instead of re-reading broad context.
  • Obsidian skills extends the same idea to a personal knowledge base, helping Claude navigate a vault through hubs, indexes, and links instead of brute-force searching.
  • The bonus Claude Code tips repo is highlighted for slash commands, problem decomposition, and proactive context compaction.
